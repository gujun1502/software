📡 商机雷达 · 一页纸工作原理

招投标商机自动筛选工具 —— 邮件进，决策日报出。全程纯 Python 骨架，大模型只在「读公告正文」时帮一把忙。

开发时 Claude 把工具「造」出来

Claude（Claude Code）在编码阶段把判断逻辑、抓取、打分、排版全部写成 Python。 成品 exe 里不含 Claude、运行时不调用 Claude——它的活在你点「运行」之前就结束了。

运行时 MiniMax 是工具里的「一个零件」

软件跑起来后，可选调用云端大模型（MiniMax / DeepSeek，OpenAI 兼容）。 本机不装任何模型，只是用 HTTP 发请求。不配也能跑（自动退回正则）。

🔍 放大第 4 步：大模型到底干了什么（只干这一件小事）

公告正文（一大段文字） → 正则先填一遍兜底 → MiniMax 读正文，抽结构化字段 → { "面积":"3500㎡", "造价":"280万元", "截标时间":"…", "关键词":[…] } → 非空字段覆盖正则值
MiniMax 何时才启动？三个闸门同时满足： ① 执行「更新数据」流程（跑到第 4 步抓详情页，只「生成日报」不碰它） ｜  ② llm_config.json 里 enabled:true 且填了 api_key（没填自动降级正则） ｜  ③ 详情页正文成功抓到。
任一不满足 → 自动退回正则，照样出日报，只是个别面积/造价可能显示「—」。这就是它在别的电脑上「死不了」的原因。

📦 换台电脑为什么能原样复刻

  • PyInstaller 把 Python 解释器+所有库塞进 exe → 对方不装 Python、不开 VPN
  • app_paths.py 让配置/data/reports 跟 exe 放一起，可直接编辑
  • 智能部分本来就在云端 → 那台机器只是换个地方发同一个 HTTP 请求

🔑 拷过去要带的东西

  • 商机雷达.exe
  • company_profile.json 公司资质（决定打分）
  • email_config.json 邮箱授权码
  • llm_config.json 填上 MiniMax api_key → 大模型就在那台机器活了
  • data/ · inbox/ 历史数据

一句话：Claude 开发期造工具（成品不含它） ·  Python 撑起 90% 确定性骨架 ·  MiniMax 运行期可选地读一段正文抽 4 个字段（云端调用，无需本地模型）。
能复刻 = 智能在云端 + exe 自带 Python + 配置跟着走 + 没网没 key 时正则永不崩。
