Out Building Decision

招投标商机雷达 + 中标决策引擎 · 系统流程设计文档

面向：工程施工 / 设计院类投标主体 数据策略：自建爬虫，采集免费公开源 本文目的：在写一行代码之前，把"这套系统每天怎么跑、决策怎么算、分几步落地"讲清楚，供你先看懂、再拍板。

一、一句话讲清这个产品

每天早上 9:15，系统自动把全网公开的招标信息扫一遍，挑出"跟我们有关"的项目，自动下载标书，建成一个项目库当天推给我；然后对每个项目，拿"我们公司的真实能力"去和"历史中标数据"对比，算出这个标我们能不能投、能排第几、该投哪些、建议怎么报价，输出一份决策报告（PDF）。

它由两个引擎组成，前者管"找"，后者管"判"：

模块 名字 职责 产出
模块一 商机雷达（Radar） 定时扫描 → 筛选 → 下载标书 → 入库 → 推送 当日商机清单 + 标书库
模块二 决策引擎（Decision） 能力匹配 → 竞争分析 → 中标概率 → 报价建议 每个项目的决策报告

两个模块串成一条流水线：雷达负责把"该看的"送到面前，决策引擎负责回答"该不该投、怎么投"。

三、模块一：商机雷达 —— 每天 9:15 自动扫描

3.1 数据从哪来（免费公开源，工程/设计类）

你选了"自建爬虫爬免费公开源"，对工程施工 / 设计院这类主体，下面这些是公开、免费、信息量大的主力源（按优先级）：

优先级 数据源 覆盖内容
★★★ 各省 / 市 公共资源交易中心（如"XX省公共资源交易网"） 工程建设、政府采购、设计招标最集中，含开标/中标结果
★★★ 中国政府采购网（ccgp.gov.cn）及 35 个省级分网 政府投资项目、设计/咨询/监理采购
★★★ 中国招标投标公共服务平台（cebpubservice.com） 全国招标公告与中标结果汇总入口
★★☆ 地方 住建 / 交通 / 水利 / 发改 主管部门网站 行业专项工程、设计任务
★★☆ 央企电子采购平台公开板块（电网、铁路、能源等） 大型工程总包/设计分包
★☆☆ 地方报建 / 施工图审查 / 招标代理机构公示页 早期商机线索，可提前布局

说明：剑鱼、采招网这类聚合站本身是付费订阅，且对爬虫有封号和合规风险，所以不爬它们，而是直接对接它们背后的"源头"——各省公共资源交易中心和政府采购网，这些是官方公开数据，合规、稳定、免费。

3.2 雷达内部五个步骤

① 采集（Fetch） - 调度器每天 9:15 触发（也支持手动"立即扫描"）。 - 每个数据源写一个"适配器"（adapter），各自知道怎么翻页、怎么取列表。 - 静态页面用普通 HTTP 抓取；有 JS 动态加载 / 验证码的，用无头浏览器（Playwright）模拟真人访问。 - 礼貌爬取：限速、随机间隔、遵守 robots、错峰，降低被封风险。

② 解析（Parse） 把每条公告抽成统一的结构化字段：

项目名称 / 采购单位 / 所在地区 / 项目类型(施工·设计·监理·EPC)
预算金额 / 资质要求 / 报名截止时间 / 开标时间 / 保证金
公告原文URL / 附件下载链接 / 发布时间

③ 筛选（Filter）—— "跟我有关"怎么定义 这是雷达的灵魂，靠一份你自定义的规则表，多条件叠加：

  • 关键词命中：行业词（如"市政道路""给排水设计""桥梁""景观"）+ 排除词（过滤无关）。
  • 地域过滤：只要本省 + 周边省，或全国（你设定）。
  • 金额区间：低于某额度不看（不值当），高于某额度也可标记"需联合体"。
  • 资质门槛预判：公告要求的资质等级，超出我方资质的直接降级或标记。
  • 时间有效性：报名已截止的自动过滤/归档。

筛选结果分三档：强相关（重点推）/ 弱相关（列入清单）/ 无关（丢弃）。

④ 下载（Download）—— 有选择地下标书 - 对"强相关 + 弱相关"的项目，自动下载招标公告全文 + 招标文件附件（pdf/doc/zip）。 - 自动重命名归档：地区_采购单位_项目名_发布日期/，附件原样保存。 - 这就形成了你要的标书下载库——不是全网无脑下，而是命中规则才下。

⑤ 入库（Store） - 结构化字段进数据库（项目表），附件进文件存储，两者用项目 ID 关联。 - 自动去重：同一项目在多个源重复发布时，按"标题+采购单位+预算"指纹合并。 - 增量更新：只处理今天新增的，历史不重复跑。

3.3 当天推送长什么样

9:15 跑完（通常几分钟到十几分钟），生成一份当日商机清单推给你：

📡 6月14日商机雷达 — 今日新增 23 条，强相关 4 条，已自动下载标书 9 份 1. 【强相关·建议投】XX区市政道路提升工程设计 · 预算 480万 · 截止 6/20 2. 【强相关·可投】XX县河道治理EPC · 预算 3200万 · 需联合体 …（每条带一键打开决策报告）

推送渠道：企业微信机器人 / 微信 / 邮件（任选）。每条商机后面挂着模块二生成的决策报告。

四、模块二：决策引擎 —— 这个标到底投不投

雷达解决"看见"，决策引擎解决"判断"。核心是拿三份数据做碰撞：

   ┌── A. 项目要求（来自标书）──┐
   │   资质门槛、业绩要求、       │
   │   规模、地域、评分办法、     │      ┌──────────────┐
   │   控制价/最高限价           │ ───▶ │   决策引擎    │ ───▶ 决策报告
   ├── B. 我方能力档案 ─────────┤      │  匹配 + 打分  │
   │   资质等级、类似业绩库、     │      │  + 概率估计   │
   │   注册资金、人员、地域、     │      └──────────────┘
   │   产能/排期                 │
   ├── C. 历史中标数据 ─────────┤
   │   同类项目谁中了、中标价、   │
   │   报价分布、常见对手        │
   └────────────────────────────┘

4.1 第一步：硬门槛过滤（废标判断）—— 先回答"有没有资格"

很多标输在还没比就被废了。引擎先做一票否决式检查，任何一条不过 = 不能投：

硬门槛 判断
资质等级 要求甲级而我方乙级 → ✗ 直接放弃（或找联合体）
类似业绩 要求"近3年2个同类业绩"，我方业绩库不足 → ✗
注册资金 / 财务 不达标 → ✗
保证金 金额、缴纳时间是否来得及 → 风险提示
报名时间 是否还来得及报名 / 制作标书 → 风险提示
废标格式陷阱 强制性格式、唯一性要求等（可联动 tender-bid-review 技能深挖）

这一步直接复用你已有的 tender-bid-review 技能的"陷阱审计"逻辑，把"设在投标界限之外的坑"提前标出来。

4.2 第二步：软匹配打分 —— 回答"我们多合适"

通过硬门槛的项目，用一个加权评分模型算"契合度"（0~100），维度针对设计院/施工：

维度 权重(示例) 怎么算
业绩相似度 30% 我方业绩库里和本项目类型/规模最接近的匹配度
资质匹配 20% 资质刚好达标=高分，远超=略降(性价比)，勉强=风险
地域优势 15% 本地/有分支/做过周边项目 → 加分（熟悉规范、运输成本低）
规模匹配 15% 项目体量与我方常规承接区间的贴合度
价格竞争力 10% 据成本结构估算我方能给的价位在市场中的位置
关系/历史 10% 与该业主是否合作过、是否有在建项目

权重你可以自己调。得分越高，越值得投入精力做标。

4.3 第三步：竞争分析 —— 回答"对手是谁、什么价能赢"

这一步用历史中标数据（C）来"反推竞争格局"：

  • 谁会来抢：同类项目历史中标人/常见投标人画像 → 估计本次潜在对手数量与强度。
  • 中标价规律：把同类历史项目的中标价/控制价做成分布，看"赢家通常压到控制价的百分之几"。
  • 评分办法识别：标书是"最低价中标"还是"综合评分"还是"均值基准价×随机K"——不同规则，最优报价策略完全不同。

报价那一步可直接调用你已有的 bid-pricing-engine 技能（博弈论 + 蒙特卡洛），输入控制价、对手分布、评分规则，算出期望收益最大的报价点和对应中标概率。

4.4 第四步：中标概率 + 报价建议

综合"契合度得分 × 竞争强度 × 报价策略"，输出量化结论：

契合度得分:        78 / 100
预计参与对手:      6~9 家
评分规则:          综合评分(技术40 + 商务60)
建议报价区间:      控制价的 92% ~ 94%（约 441万~451万）
对应中标概率:      约 34%
预期收益(概率×毛利): ★★★☆☆
决策标记:          ✅ 建议投

4.5 第五步：决策报告（最终产出 · PDF）

每个值得看的项目，生成一页式决策报告，结构如下：

┌────────────────────────────────────────────────────┐
│  项目决策报告 · XX区市政道路提升工程设计              │
├────────────────────────────────────────────────────┤
│ 一、项目速览   预算/截止/业主/地区/项目类型           │
│ 二、资格体检   硬门槛逐条 ✓/✗，废标风险提示           │
│ 三、契合度     雷达图 + 各维度得分                     │
│ 四、竞争格局   预计对手、历史中标价分布、评分规则       │
│ 五、报价建议   建议区间 + 中标概率 + 收益评级          │
│ 六、决策结论   建议投 / 可投 / 放弃 + 一句话理由       │
│ 七、行动清单   报名截止、需准备的业绩/资质材料         │
└────────────────────────────────────────────────────┘

多个项目时，再叠加一张当日排序总表：把所有"建议投/可投"的项目按"预期收益"从高到低排序，让你一眼看出"今天先抢哪几个"。这就直接回答了你说的"我方可能赢得的标的排序跟范围"。

六、数据模型（三张核心表）

1. 项目表（雷达产出） 项目ID · 名称 · 业主 · 地区 · 类型 · 预算 · 资质要求 · 业绩要求 · 截止时间 · 开标时间 · 保证金 · 原文URL · 附件路径 · 相关度等级 · 抓取日期

2. 我方能力档案（你维护一次，长期复用） 资质清单(类别/等级/有效期) · 业绩库(项目名/类型/规模/年份/业主/角色) · 注册资金 · 关键人员/职称 · 可服务地域 · 常规承接规模区间 · 成本结构参数

3. 历史中标数据库（持续积累） 项目类型 · 地区 · 控制价 · 中标价 · 中标人 · 投标家数 · 评分办法 · 年份 —— 雷达每天抓"中标公示"自动回填，越用越准。

七、每天 9:15 的完整时序

09:15  调度触发，启动各源适配器
09:15  并发采集今日新公告（限速、错峰）
09:18  解析 → 抽取结构化字段
09:19  规则筛选 → 强/弱/无关三档
09:20  对强+弱相关项目下载标书附件、归档入库
09:22  决策引擎：硬门槛 → 软匹配 → 竞争分析 → 报价
09:25  生成每个项目决策报告 + 当日排序总表(PDF)
09:26  推送当日商机清单到企业微信/邮件
09:26  ✅ 完成，你打开手机就能看

全程无人值守。你要做的只有两件事：(1) 一次性配好规则和能力档案；(2) 每天早上看推送、拍板。

八、落地路线图（分三期，先能用再做强）

阶段 目标 内容 大致周期
第一期 MVP 跑通"扫描→下载→推送" 1~2 个核心数据源(本省公共资源交易中心) + 关键词筛选 + 每日 PDF 清单推送。决策引擎先做硬门槛+简单打分 2~3 周
第二期 决策引擎做深 接入能力档案、历史中标库、bid-pricing-engine 报价模型，输出完整决策报告 3~4 周
第三期 扩源 + 提准 增加数据源到 5~8 个，去重/反爬加固，中标数据自动回填，权重自学习 持续迭代

建议：先用第一期验证"雷达每天能不能稳定把对的项目送到你面前"，这是整个产品价值的地基；地基稳了，决策引擎是锦上添花、越用越准。

九、合规与风险提示（自建爬虫必看）

  1. 只爬公开信息：政府采购网、公共资源交易中心的招标/中标公示属公开政务信息，采集合规；不爬付费聚合站、不绕付费墙、不抓个人隐私。
  2. 礼貌爬取：遵守 robots.txt、限速、错峰、不高频冲击源站，避免被认定为攻击。
  3. 数据仅自用：抓取结果用于自身投标决策，不二次售卖/公开传播。
  4. 反爬应对要稳：部分源有验证码/动态加载，用无头浏览器但保持低频；个别源若反爬强，宁可降频也不硬刚。
  5. 决策仅供参考：中标概率是模型估计，不是承诺；最终投标决策由人拍板，报告是"辅助驾驶"不是"自动驾驶"。

十、这份文档之后，怎么往下走

你看完这套流程后，我们可以按你的反馈： - 调整规则：把第三节的关键词、地域、金额门槛换成你们真实的业务画像； - 填能力档案：把第六节的资质/业绩/规模填成你公司的真实数据； - 选数据源：确定先接哪个省/市的公共资源交易中心做第一期 MVP； - 开工：按第八节路线图，从第一期 MVP 开始写代码。

一句话总结这套系统的价值：把你每天本来要花 1~2 小时翻招标网、人工判断"投不投"的活，压缩成早上一条推送 + 一份排好序的决策报告。
